# Vue 前端开发与编码规范 (vue_conventions.md)

## 1. 组件开发与命名规范
*   **文件命名**：Vue 组件的文件名必须采用 **大驼峰命名 (PascalCase)**，且禁止使用单单词命名（例如使用 `UserManage.vue` 代替 `User.vue`），以符合 Vue 风格指南，防止与底层 HTML 标签冲突。
*   **组件开发风格**：本项目基于 Vue 2.7.x，统一采用 **Options API** 编写逻辑，数据声明在 `data()` 中，方法写入 `methods`。
*   **单一职责与规模限制**：保持组件短小精悍。一旦单个组件的模板和逻辑代码行数**超过 300 行**，必须考虑进行模块拆分和子组件提取。

---

## 2. 样式隔离规范 (CSS)
*   **局部样式作用域**：在组件内编写样式时，必须在 `<style>` 标签上声明 **`scoped`** 属性（如 `<style scoped>`），以实现局部样式哈希隔离，防止样式全局污染。
*   **样式定制**：如果需要覆写 Element UI 内部组件的样式，必须使用深度作用选择器 `/deep/` 或 `::v-deep`。

---

## 3. 网络请求与网络拦截规范 (Axios)
*   **API 统一管理**：所有的网络请求必须在 `src/api/` 下按照业务模块集中声明，组件内部直接导入使用，禁止在 Vue 组件 methods 中直接编写 raw `axios.get/post`。
*   **请求拦截**：在 `request.js` 中拦截请求，自动从 LocalStorage 读取 Token，并装载进 Header 的 `token` 或 `Authorization` 字段。
*   **响应拦截**：
    *   网络拦截器在识别到后端 Result 返回 `code = 0` 时，统一给出用户警示。
    *   在捕获到 `401` (未登录/会话过期) 错误时，清空本地 LocalStorage (清除 Token、Role 等)，并使用路由强制重定向回 `/login` 登录页面。

---

## 4. 路由与菜单权限控制
*   **路由元数据**：在定义路由时，对受保护的页面增加 `requiresAuth: true`。需要特定角色访问的页面配置 `roles: [3]` 数组。
*   **全局路由守卫**：路由守卫在每次导航前对 Token 及 Role 进行比对，不满足权限时重定向至 `/403` 页面。
