# 设备管理系统四级角色定位与职责矩阵

本设计文档旨在明确设备管理系统中 **0-设备操作员、1-维修工程师、2-资产管理员、3-系统管理员 (超级运维)** 四个角色的业务定位、核心职责以及各功能模块的权限控制细则。

---

## 👥 角色业务定位与核心职责

### 0 - 设备操作员 (普通员工)
*   **角色定位**：设备的最终领用人、使用者和第一报修发起人。
*   **核心职责**：
    *   妥善保管并日常使用分派给自己保管的设备。
    *   在自己领用的设备发生故障时，及时在系统内发起报修，并跟进工单的维修进度。
*   **权限原则**：**最小可用原则 + 严格数据隔离**。仅能查看和操作与自身直接绑定的设备和工单，其余管理功能均不可见。

### 1 - 维修工程师
*   **角色定位**：设备检修和故障排除的专业技术人员（执行层）。
*   **核心职责**：
    *   接收系统分配的设备维修工单。
    *   负责现场故障排查与修复，并在系统内如实登记“维修结果”（包括维修内容、材料费用、维修人工等信息）。
*   **权限原则**：**业务只读 + 检修可写**。可以全局查看设备台账了解技术参数，但只对检修单有处理权，无任何资产调拨、报废或系统管理权限。

### 2 - 资产管理员
*   **角色定位**：设备资产全生命周期的**业务运营管理者**（管理层）。
*   **核心职责**：
    *   负责资产“入库、分配、流转、报废”的全链路管理。
    *   担任维修“派单人”，负责审核报修工单并指派具体的维修工程师。
    *   维护设备分类、使用期限、公司组织架构等业务主数据。
*   **权限原则**：**业务全特权 + 系统只读/隐藏**。在设备、调拨、报废、检修、分类、单位模块拥有最高的写权限，但无系统级管理功能（如修改用户角色、数据库备份恢复）。

### 3 - 系统管理员 (超级运维)
*   **角色定位**：系统的安全运营、基础技术保障与审计者（运维层）。
*   **核心职责**：
    *   负责系统账号生命周期管理，分配并调整各岗位员工的角色。
    *   负责数据库安全管理，支持手动 SQL 备份与一键灾备还原。
    *   扮演“超级审计”角色，对所有业务数据拥有只读视界，但不直接操作具体业务。
*   **权限原则**：**系统特权 + 业务审计只读**。独占用户管理和备份还原写权限，业务数据全局只读，实现管理与运营权责分离。
