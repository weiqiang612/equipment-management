# 国家标准设备管理系统 (EMS) - 原型与 UI 设计规范 (ui_prototype.md)

## 1. 主视觉与色彩规范
本系统采用 **科技工业蓝灰** 视觉体系，界面要求清晰、现代且符合国家标准软件的使用习惯。
*   **主色 (Primary)**：`#409EFF` (Element UI 经典蓝)
*   **成功色 (Success)**：`#67C23A` (用于“在用”设备状态、备份成功标识)
*   **警告色 (Warning)**：`#E6A23C` (用于“维修”设备状态、恢复确认提示)
*   **危险色 (Danger)**：`#F56C6C` (用于“报废”设备状态、删除设备、清除 Token 退出)
*   **侧边栏背景**：`#304156` (科技暗蓝灰)

---

## 2. 界面布局结构
系统采用经典的后台管理左右分栏布局：
1.  **左侧导航栏 (Aside)**：
    *   折叠/展开宽度：200px。
    *   **RBAC 动态隐藏**：依据当前用户的 `role` 渲染菜单。操作员只可见“我的设备”、“报修申请”；其他角色可见“设备资产管理”、“分类管理”、“单位管理”等。
2.  **右侧主区域**：
    *   头部 (Header)：面包屑导航、用户真实姓名欢迎词及当前角色 Tag 标签、退出登录按钮。
    *   主视图 (Main)：背景色 `#f0f2f5`，留白 20px，内部使用卡片组件 (`el-card`) 承载表格和检索表单。

## 4. UI 组件像素级还原准则
*   **组件命名**：Vue 组件文件命名一律使用大驼峰且不使用单个单词（如 `UserManage.vue` 而非 `User.vue`）。
*   **自适应设计**：表格和表单必须支持在 1280px 至 1920px 宽度下的自适应缩放，超出部分使用分页器拦截，禁止页面底端出现横向滚动条。
*   **操作反馈**：
    *   删除、恢复、退出等敏感操作必须使用 `this.$confirm` 弹出危险提示确认框。
    *   接口交互成功后必须触发 `this.$message.success` 进行吐司提示。

## 5. 数据看板 (Dashboard) UI 规范 (新)
*   **排版栅格系统**：使用 `el-row` 和 `el-col`（结合 `span` 或 `:xs`, `:sm`, `:md`, `:lg`）构建自适应网格。指标卡片在 1920px 屏幕下每行 4~5 列，在 1280px 下换行或堆叠。
*   **指标卡片 (KPI Cards)**：采用白色卡片背景，左侧为彩色状态圆角图标，右侧为大字号数字指标（如原值使用货币符号 `￥` 并保留 2 位小数），数字下方为小字号灰色指标名称。
*   **图表卡片**：ECharts 图表需包裹于固定高度（如 `350px`）的 `div` 容器内。
*   **图表交互与自适应**：
    - 图表初始化必须在组件 `mounted` 声明周期进行，并在 `beforeDestroy` 时销毁实例，防内存泄漏。
    - 绑定 `window.addEventListener('resize', this.handleResize)` 以在视口变化时调用 `chart.resize()` 自适应重绘。
*   **待办工作台表格**：看板下方的“待办”或“我的设备”等列表直接复用简洁的 `el-table`，但去除复杂的搜索过滤区，以最直观的信息列配合 “处理” 跳转按钮（使用 `cursor-pointer`）形式展现。

## 6. 数据治理 (Governance) UI 规范 (新)
*   **入口权限**：左侧导航新增“数据治理”或“风险分析”入口，仅 Role 2 资产管理员和 Role 3 系统管理员可见。
*   **页面结构**：
    - 顶部使用 KPI 卡片展示数据质量问题数、高风险设备数、中风险设备数、长期空闲设备数和维修成本异常数。
    - 中部使用图表或进度条展示风险等级分布、部门风险分布和分类风险分布。
    - 下方使用 `el-table` 展示风险设备清单，支持风险等级、单位、分类筛选。
*   **状态色**：
    - 高风险使用危险色 `#F56C6C`。
    - 中风险使用警告色 `#E6A23C`。
    - 低风险使用成功色 `#67C23A` 或信息色。
    - 数据质量问题使用警告标签，并在详情列说明具体原因。
*   **交互状态**：
    - 页面加载时使用 `v-loading` 或骨架状态。
    - 无风险数据时展示空状态，不显示空白页面。
    - 接口失败时展示错误提示和重试入口。
*   **响应式要求**：1280px 宽度下 KPI 卡片自动换行；风险设备表格保持横向信息密度，必要时通过分页和列宽控制避免页面底部横向滚动。

## 7. AI 辅助 (AI Assistant) UI 规范 (新)
*   **入口权限**：左侧导航新增“AI 助手”或“AI 报告”入口，仅 Role 2 资产管理员和 Role 3 系统管理员可见。
*   **页面结构**：
    - 顶部展示 AI 功能边界提示，明确“AI 输出仅为草案，需人工确认”。
    - 左侧提供报告类型选择：周报、月报、设备生命周期摘要。
    - 右侧展示生成结果，包括报告正文、风险证据、建议动作和复制按钮。
*   **交互状态**：
    - 生成中使用 `v-loading` 和按钮 loading，防止重复提交。
    - Provider 未配置时展示“AI 服务未配置”的空状态和处理建议。
    - Provider 调用失败时展示错误提示和重试按钮。
*   **安全呈现**：
    - 所有 AI 文本区域必须带“草案”或“建议”标识。
    - 不提供“自动审批”“自动报废”“自动恢复数据库”等高风险按钮。
    - 复制草案只复制文本，不触发任何业务写操作。
