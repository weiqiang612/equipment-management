# TASK-006: 前端数据隔离与操作隔离

**Status**: Draft
**Created**: 2026-06-10
**Feature dir**: `docs/3-tasks/features/TASK-006-frontend-isolation/`

## Objective
完善前端用户界面的角色准入、数据隔离以及操作隔离设计。配合后端升级，在界面中对高权操作、非关联的记录操作以及跨部门侧边栏入口进行彻底隔离或限制；并将工单指派输入方式重构为从维修工列表中选择，规避手填安全风险。

## Scope

### In scope

#### 【P0 级核心操作与路由隔离】
1. **路由强拦截**：收紧调拨记录、报废记录、分类管理与单位管理路由的准入角色 `roles` 限制为 `[2, 3]`，维修工（1）或操作员（0）越权访问时直接路由拦截重定向到 `/403`。
2. **菜单动态显隐**：调整 `App.vue` 侧边栏，为维修工（1）和操作员（0）隐藏调拨记录、报废记录、分类管理和单位管理菜单。
3. **操作列与新增按钮隐藏**：
   - 针对系统管理员（3），在设备台账和检修记录页面，**整体使用 `v-if` 隐藏“操作”整列**，并隐藏“新增设备”、“新增检修补录”等写操作按钮，确保其“审计只读”属性。
   - 设备台账中，操作员（0）仅在操作列中保留“维修”按钮（报修登记），其余编辑、删除、调拨、报废等按钮均隐藏。
4. **同角色工单数据修改隔离**：在检修记录列表中，维修工（1）仅可见自己作为检修人（`maintPerson` 与当前登录的 `realName` 一致）的修改按钮，对于其他工程师的检修记录，隐藏其“修改”与“删除”动作。

#### 【P1 级接口与指派组件对接】
5. **指派维修工下拉选择器**：将原本手动输入维修工姓名的输入框，改造为 `el-select` 下拉选择器，且该下拉列表数据必须是通过请求后端获取的**所有角色为 `1` (维修工程师) 的用户列表**。
6. **调拨与报废操作列控制**：设备台账操作列中的“编辑”、“删除设备”、“调拨”与“报废”选项，仅限角色 `2` (资产管理员) 可见并点击。

### Out of scope
- 后端控制器的 AOP 鉴权、BaseContext 建立与 SQL 数据过滤逻辑（已在后端任务 TASK-005 中实现）。
